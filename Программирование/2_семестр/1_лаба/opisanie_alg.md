    <h1>Описание алгоритма функции <code>read_words_to_array</code></h1>
    <p>Функция <code>read_words_to_array</code> считывает слова из текстового файла и сохраняет их в упорядоченной структуре данных. Разберём алгоритм пошагово.</p>
    <h2>Входные и выходные данные</h2>
    <h3>Входные параметры:</h3>
    <ul>
        <li><code>filename</code> — имя файла для чтения;</li>
        <li><code>top_size</code> — указатель на переменную, куда будет записано количество блоков в результате.</li>
    </ul>
    <h3>Возвращаемое значение:</h3>
    <ul>
        <li>указатель на трёхмерный массив (<code>char***</code>) — структура данных, содержащая отсортированные слова;</li>
        <li><code>NULL</code>, если файл не удалось открыть.</li>
    </ul>
    <h2>Шаг 1. Инициализация и проверка файла</h2>
    <ol>
        <li>Открываем файл на чтение с помощью <code>fopen</code>.</li>
        <li>Если файл не удалось открыть, устанавливаем <code>*top_size = 0</code> и возвращаем <code>NULL</code>.</li>
    </ol>
    <h2>Шаг 2. Подготовка структуры данных</h2>
    <ol>
        <li>Выделяем память для «верхнего» массива (<code>top</code>) с начальной ёмкостью <code>cap = 10</code> блоков.</li>
        <li>Инициализируем <code>*top_size = 0</code> (количество блоков пока равно нулю).</li>
        <li>Создаём буфер <code>word[256]</code> для временного хранения считываемого слова.</li>
    </ol>
    <h2>Шаг 3. Пословное чтение файла</h2>
    <p>В цикле <code>while</code> считываем слова из файла с помощью <code>fscanf(f, "%255s", word)</code>:</p>
    <ul>
        <li>каждое слово копируется в динамически выделенную память (<code>new_word</code>);</li>
        <li>далее слово нужно вставить в структуру данных с сохранением сортировки.</li>
    </ul>
    <h2>Шаг 4. Поиск позиции для вставки</h2>
    <p>Для нового слова ищем позицию <code>pos</code> в «верхнем» массиве <code>top</code>:</p>
    <ul>
        <li>сравниваем <code>new_word</code> с первым словом каждого блока (<code>top[pos][0]</code>) с помощью <code>strcmp</code>;</li>
        <li>позиция <code>pos</code> — это индекс блока, куда должно попасть новое слово (или в новый блок в конце, если слово больше всех существующих).</li>
    </ul>
    <h2>Шаг 5. Вставка в существующий блок</h2>
    <p>Если найденный блок (<code>pos < *top_size</code>) существует:</p>
    <ol>
        <li>Считаем количество слов в блоке (<code>cnt</code>), ища <code>NULL</code>‑маркер.</li>
        <li>Если в блоке есть место (<code>cnt < BOTTOM_SIZE</code>):
            <ol>
                <li>находим позицию <code>ins</code> внутри блока для сохранения сортировки;</li>
                <li>сдвигаем слова вправо, чтобы освободить место;</li>
                <li>вставляем <code>new_word</code>.</li>
            </ol>
        </li>
        <li>Если блок переполнен (<code>cnt == BOTTOM_SIZE</code>):
            <ol>
                <li>делим блок пополам:
                    <ol>
                        <li>создаём новый блок (<code>new_block</code>) для второй половины слов;</li>
                        <li>копируем вторую половину слов (<code>half = BOTTOM_SIZE / 2</code>) в новый блок;</li>
                        <li>обрезаем текущий блок, устанавливая <code>NULL</code> после первой половины;</li>
                    </ol>
                </li>
                <li>расширяем «верхний» массив <code>top</code>, если нужно (удваиваем <code>cap</code> и <code>realloc</code>);</li>
                <li>сдвигаем блоки в <code>top</code> вправо, чтобы вставить новый блок;</li>
                <li>увеличиваем <code>*top_size</code>;</li>
                <li>определяем, в какой блок вставить <code>new_word</code> (в первый или второй), и вставляем его с сохранением порядка.</li>
            </ol>
        </li>
    </ol>
    <h2>Шаг 6. Создание нового блока</h2>
    <p>Если нужно добавить слово в конец (<code>pos == *top_size</code>):</p>
    <ol>
        <li>При необходимости расширяем <code>top</code> (удваиваем <code>cap</code> и <code>realloc</code>).</li>
        <li>Создаём новый блок <code>new_block</code> с ёмкостью <code>BOTTOM_SIZE + 1</code> (последний элемент — <code>NULL</code>).</li>
        <li>Помещаем <code>new_word</code> в начало блока, остальные элементы инициализируем как <code>NULL</code>.</li>
        <li>Добавляем блок в <code>top</code> и увеличиваем <code>*top_size</code>.</li>
    </ol>
    <h2>Шаг 7. Завершение</h2>
    <ol>
        <li>Закрываем файл <code>fclose(f)</code>.</li>
        <li>Возвращаем указатель на структуру <code>top</code>.</li>
    </ol>
    <h2>Структура данных</h2>
    <ul>
        <li><strong>«Верхний» массив (<code>top</code>)</strong> — массив указателей на блоки (<code>char**</code>). Хранит блоки слов в отсортированном порядке.</li>
        <li><strong>Блоки (<code>char**</code>)</strong> — массивы указателей на слова. Каждый блок содержит до <code>BOTTOM_SIZE</code> слов, отсортированных по алфавиту. Последний элемент блока — <code>NULL</code>.</li>
        <li><strong>Слова (<code>char*</code>)</strong> — строки, выделенные динамически.</li>
    </ul>
    <h2>Ключевые особенности</h2>
    <ul>
        <li><strong>Сортировка:</strong> слова хранятся в алфавитном порядке внутри блоков и между блоками.</li>
        <li><strong>Динамическое расширение:</strong> «верхний» массив удваивается при нехватке места (<code>realloc</code>).</li>
        <li><strong>Балансировка:</strong> при переполнении блока он делится пополам, чтобы сохранить эффективность вставки.</li>
        <li><strong>Экономия памяти:</strong> используется блочная структура вместо того, чтобы хранить все слова в одном большом массиве.</li>
    </ul>
    <h2>Пример работы</h2>
    <p>Допустим, <code>BOTTOM_SIZE = 3</code>. После вставки слов «cat», «dog», «ant», «bat»:</p>
    <pre>
top[0]: ["ant", "bat", "cat", NULL]
top[1]: ["dog", NULL, NULL, NULL]
    </pre>
    <p>При вставке «emu» блок <code>top[1]</code> переполнится, и он будет разделён:</p>
    <pre>
top[0]: ["ant", "bat", "cat", NULL]
top[1]: ["dog", NULL, NULL, NULL]
top[2]: ["emu", NULL, NULL, NULL]
    </pre>
